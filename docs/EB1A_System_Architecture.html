Booking-to-Payment Reconciliation Engine

NOVEL Square's APIs have no foreign key between Bookings and Payments. This engine solves an industry-wide unsolved problem:

Phase 1: API-Based Reconciliation 1. Fetch booking details from Square Bookings API 2. Extract service types from booking segments 3. Fetch payment line items from Square Payments API 4. Calculate service-type overlap score (Jaccard similarity) 5. If score > threshold → MATCH Phase 2: Database Temporal Fallback 1. Match by customer_id + location_id 2. Filter by time proximity (booking end_at ± 2 hours of payment created_at) 3. Rank by temporal distance 4. If unique match found → MATCH Result: Links every appointment to its payment transaction → Enables accurate per-service, per-staff revenue attribution → No existing platform can do this
